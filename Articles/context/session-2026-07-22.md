## Critical Mobile Bug — Fixed (21 July)

**Root cause**: `<nav class="mobile-bottom-nav">` inherits `top:0` from base `nav { position:sticky; top:0 }` CSS. With `bottom:0; position:fixed` but NO `top:auto`, element stretched full viewport → solid dark background covered page → blank screen on iOS.

**Fix**: `<div class="mobile-bottom-nav" role="navigation">` on ALL pages.
**Secondary fix**: Remove `backdrop-filter:blur()` from `position:fixed` elements (iOS compositing bug). Use solid `background:#08121f`.
**Tertiary fix**: Mobile CSS must be in `<head>`, never at bottom of `<body>`.

## Mobile Patterns (Always Follow These)

```css
/* Mobile bottom nav — ALWAYS use <div>, never <nav> */
.mobile-bottom-nav {
  display: none;
  position: fixed; bottom: 0; left: 0; right: 0; z-index: 300;
  background: #08121f;  /* solid, NO backdrop-filter */
  border-top: 1px solid rgba(245,158,11,0.25);
  padding: 8px 0 max(8px, env(safe-area-inset-bottom));
}
@media(max-width:768px) { .mobile-bottom-nav { display:block; } body { padding-bottom:68px; } }
/* CSS MUST be in <head>, never in <body> */
```

## Chart.js PDF Export Pattern (both planners)

```javascript
// Use a separate hidden canvas for print — do NOT use the live chart canvas
const pc = document.getElementById('printChartCanvas'); // 860x480, position:absolute left:-9999px
// Build fresh Chart.js instance with responsive:false
// Export via offscreen canvas with white background:
const offscreen = document.createElement('canvas');
offscreen.width = pc.width; offscreen.height = pc.height;
const ctx2 = offscreen.getContext('2d');
ctx2.fillStyle = '#ffffff';
ctx2.fillRect(0,0,offscreen.width,offscreen.height);
ctx2.drawImage(pc,0,0);
document.getElementById('printChartImg').src = offscreen.toDataURL('image/png');
setTimeout(() => window.print(), 250);
```
