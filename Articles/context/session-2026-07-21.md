## Mobile Bug Fixes Applied (July 21 session)

### Root cause of blank screens on tools.html and learn.html:
1. **`backdrop-filter:blur(16px)` on `position:fixed`**: Known iOS Safari bug — a fixed element with backdrop-filter causes the page content behind it to go blank. Fixed by removing backdrop-filter from `.mobile-bottom-nav` (replaced with solid `background:#08121f`) and from `.nav-links` dropdown.
2. **Late `<style>` tag in learn.html body**: The mobile bottom nav CSS was added at the bottom of `<body>` (after `<footer>`). Mobile browsers can fail to render when `<style>` tags appear in body. Fixed by moving all CSS to `<head>`.

### Pattern for mobile nav (all pages) — after fix:
```css
/* NO backdrop-filter on position:fixed or position:absolute nav elements */
.mobile-bottom-nav { background: #08121f; /* solid, no blur */ }
.nav-links { background: #08121f; /* solid, no blur */ }
```

## Architecture / Technical Notes

### Chart.js PDF Export Pattern
```javascript
// In generatePDF() — do NOT use the live canvas
// Create a separate hidden canvas:
const pc = document.getElementById('printChartCanvas'); // 860x480, position:absolute left:-9999px
// Build fresh Chart.js instance on it with responsive:false
// Then export via offscreen canvas with white background fill:
const offscreen = document.createElement('canvas');
offscreen.width = pc.width; offscreen.height = pc.height;
const ctx2 = offscreen.getContext('2d');
ctx2.fillStyle = '#ffffff';
ctx2.fillRect(0, 0, offscreen.width, offscreen.height);
ctx2.drawImage(pc, 0, 0);
document.getElementById('printChartImg').src = offscreen.toDataURL('image/png');
setTimeout(() => window.print(), 250);
```

### PDF Print CSS Pattern
```css
@page { size: A4; margin: 0; }
/* hide all web UI, show only .print-only divs */
/* pages use page-break-after: always */
/* cover uses flexbox min-height:289mm to pin footer */
```

### Paywall Pattern
```javascript
const unlocked = new URLSearchParams(window.location.search).get('unlocked') === 'true';
// blur + overlay on paid sections if !unlocked
// PDF button shows Stripe link if !unlocked
```

### Mobile Bottom Nav Pattern (all three pages)
```css
/* IMPORTANT: No backdrop-filter on fixed/absolute nav elements — iOS Safari bug */
.mobile-bottom-nav { display:none; position:fixed; bottom:0; z-index:300; background:#08121f; border-top:1px solid rgba(245,158,11,0.25); padding:8px 0 max(8px,env(safe-area-inset-bottom)); }
@media(max-width:768px) { .mobile-bottom-nav { display:block; } body { padding-bottom:68px; } }
/* CSS MUST be in <head>, NOT in a <style> tag at the bottom of <body> */
```

### Hamburger Nav Pattern
```css
.nav-hamburger { display:none; } /* shown on mobile */
.nav-links { display:flex; } /* hidden on mobile, absolute dropdown when .mobile-open */
/* On mobile: display:none → display:flex when .mobile-open added by JS */
```
